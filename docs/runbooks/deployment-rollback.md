# Deployment rollback runbook

Use rollback only when the currently deployed contract set is actively breaking users and a fast restore to the last known-good manifest is safer than a forward fix.

Prefer a forward fix when:

- the issue is isolated to frontend code,
- contract state has already moved forward and older WASM cannot safely operate on it,
- a hotfix can be deployed faster than validating the previous manifest,
- there is no known-good `deployments/v1/<network>.previous.json` file.

Use rollback when:

- a newly deployed WASM hash is causing transaction failures,
- the manifest points clients at a bad contract ID,
- RPC smoke tests pass but contract invocations fail immediately after deployment,
- the previous deployment is known to be compatible with current state.

## Prerequisites

1. Confirm the previous manifest exists:

```bash
ls deployments/v1/testnet.previous.json
```

If the file does not exist yet, copy the template and fill it with the last validated contract IDs and `wasmHash` values:

```bash
cp deployments/v1/testnet.previous.example.json deployments/v1/testnet.previous.json
```

2. Confirm Stellar CLI is authenticated for the target network when using `--execute`.
3. Confirm the previous manifest contains the known-good contract IDs and `wasmHash` values.
4. Announce the rollback window to operators.
5. Capture the current manifest SHA or copy the active manifest before rollback.

## Dry run

Always start with dry run:

```bash
node scripts/rollback-deployment.mjs --network testnet --dry-run
```

The dry run prints structured rollback evidence:

- active manifest path,
- previous manifest path,
- active deployment ledger,
- rollback deployment ledger,
- contracts with known-good WASM hashes,
- Stellar CLI deployment commands that would run,
- suggested smoke test commands.

Do not continue if the dry-run output references an unexpected network, empty contract list, or unknown previous manifest.

## Execute rollback

```bash
node scripts/rollback-deployment.mjs --network testnet --execute --smoke
```

The script will:

1. Back up the active manifest with a timestamp suffix.
2. Restore `deployments/v1/<network>.json` from `deployments/v1/<network>.previous.json`.
3. Run deployment commands for every contract with a known-good `wasmHash`.
4. Run post-rollback smoke tests when `--smoke` is passed.

## Post-rollback smoke tests

Run these manually if `--smoke` was not used:

```bash
node scripts/verify-deployment-manifest.mjs --network testnet --strict
node scripts/probe-horizon-rpc.mjs --network testnet
cd frontend && npm run build
```

Then verify from the app:

1. Wallet connects on the target network.
2. Schema read/list flows load.
3. Attestation issue flow prepares a transaction.
4. Revocation management prompts Freighter and surfaces a clear double-revoke error if retried.
5. Schema deprecation prompts Freighter, updates the local schema state, and displays the deprecated badge.

## Recovery notes

- Keep the backup manifest generated by the script until the incident is closed.
- Open a follow-up issue for the forward fix that replaces the rollback.
- Record the rollback transaction hashes, manifest diff, probe output, and smoke-test output in the incident notes.
- If rollback fails, stop further deploy attempts, preserve logs, and escalate to a forward-fix incident branch.
